Выполни self-review последних изменений.

## Процесс

### 1. Определи scope изменений
- Запусти `git diff` для unstaged и `git diff --cached` для staged
- Если нет diff — проверь $ARGUMENTS как список файлов

### 2. Проверка на каждый изменённый файл

Для каждого файла задай 5 вопросов:

1. **Корректность**: делает ли код то, что задумано? Все ли ветки обработаны?
2. **Типизация**: есть ли неявные `any`? Правильно ли работают null/undefined?
3. **Ошибки**: что произойдёт при сетевом сбое? При пустых данных? При невалидном вводе?
4. **Побочные эффекты**: не сломает ли это другие подсистемы? Работает ли бот без DATABASE_URL?
5. **Чистота**: есть ли мёртвый код, лишние импорты, debug-логи?

### 3. Найди минимум 3 улучшения
Даже если код выглядит идеально — найди 3 момента:
- Можно ли улучшить именование?
- Можно ли упростить условие?
- Можно ли добавить guard clause?
- Можно ли уточнить тип?
- Можно ли улучшить сообщение об ошибке?

### 4. Исправь найденные замечания
Не откладывай — исправь сразу. После исправления:
- `npx tsc --noEmit`
- `npm run build`

### 5. Финальный отчёт
- Список найденных и исправленных замечаний
- Оценка готовности: **Ready to commit** / **Needs more work**
