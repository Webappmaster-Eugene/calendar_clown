Выполни code review для $ARGUMENTS:

## Шаги

1. Прочитай указанные файлы или `git diff` (если $ARGUMENTS пуст — проверь uncommitted changes)
2. Проверь каждый из аспектов ниже и выведи результат в формате чеклиста

## Чеклист проверки

### Type Safety
- [ ] Нет `any`, `as any`, `@ts-ignore` без обоснования
- [ ] Параметры функций и возвращаемые значения типизированы явно
- [ ] Корректное использование `null`/`undefined` (strict null checks)

### Обработка ошибок
- [ ] Все async-операции обёрнуты в try/catch
- [ ] Пользователю отправляется понятное сообщение об ошибке (ctx.reply)
- [ ] Ошибки логируются в console.error с контекстом

### Безопасность
- [ ] Нет захардкоженных секретов (токены, ключи)
- [ ] Входные данные валидируются (user input из Telegram)

### Архитектура
- [ ] Stateless-подход (нет скрытого глобального состояния)

### Стиль кода
- [ ] ES Module imports с `.js` расширением
- [ ] Двойные кавычки, 2 пробела отступ
- [ ] camelCase для переменных, PascalCase для типов
- [ ] Нет мёртвого кода, закомментированных блоков, TODO без issue

### Производительность
- [ ] Нет утечек памяти (потоки закрыты, таймеры очищены)
- [ ] Временные файлы удаляются после использования (voice/)
- [ ] Нет блокирующих синхронных операций

## Формат вывода

Для каждого найденного замечания укажи:
- **Файл:строка** — конкретное место
- **Серьёзность** — critical / warning / suggestion
- **Описание** — что не так и почему
- **Исправление** — как исправить (с примером кода, если уместно)

В конце — краткое резюме: сколько замечаний по каждой серьёзности.
