Проверь готовность проекта к деплою.

## Чеклист

### 1. Код
- Запусти `npx tsc --noEmit` — нет ошибок типизации
- Запусти `npm run build` — сборка проходит
- Проверь `git status` — нет случайных файлов (.env, data/, deploy_key)
- Проверь `git diff --cached` — в коммите только нужные изменения

### 2. Конфигурация
- Прочитай `.github/workflows/deploy.yml` — workflow актуален
- Проверь, что `docker-compose.yml` синтаксически корректен
- Проверь nginx конфиги в `config/` — нет синтаксических ошибок
- Убедись, что `scripts/ensure-oauth-ssl.sh` не менялся без необходимости

### 3. Миграции
- Если есть новые миграции в `migrations/` — убедись, что они идемпотентны
- SQL-запросы используют `IF NOT EXISTS` / `IF EXISTS` где применимо

### 4. Переменные окружения
- Если добавлены новые env — обновлены ли README.md, .env.example?
- Новые секреты добавлены в описание GitHub Secrets в README?
- Бот продолжает работать без опциональных env (DATABASE_URL, OPENCLAW_*)

### 5. Безопасность
- Нет секретов в коде (grep на `token`, `password`, `secret`, `key` в src/)
- `.gitignore` содержит все чувствительные файлы
- OAuth redirect использует HTTPS

### 6. Результат
Выведи итоговый статус:
- **READY** — всё ок, можно деплоить
- **BLOCKED** — список проблем, которые нужно исправить
